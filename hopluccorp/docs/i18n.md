# Đa ngôn ngữ (i18n) — Quyết định & Thiết kế

> Cập nhật: 2026-03-08

## Tổng quan

Tất cả nội dung trên trang hỗ trợ **4 ngôn ngữ**. Cả text lẫn hình ảnh đều có thể thay đổi theo ngôn ngữ.

| Ngôn ngữ | Code | Vai trò |
|----------|------|---------|
| Tiếng Việt | `vi` | **Mặc định + Fallback** |
| English | `en` | — |
| 中文 (Chinese) | `zh` | — |
| 한국어 (Korean) | `ko` | — |

**Fallback**: Nếu bản dịch EN/ZH/KO chưa có → tự động hiển thị bản VI.

---

## Package: `django-modeltranslation`

### Tại sao chọn?

| Tiêu chí | modeltranslation | parler | JSONField |
|----------|:-:|:-:|:-:|
| Admin UX (tabs ngôn ngữ) | Tốt nhất | Trung bình | Kém |
| CKEditor 5 tương thích | Hoàn toàn | Hạn chế | Không |
| ImageField hỗ trợ | Hoàn toàn | Hoàn toàn | Không |
| Không sửa code cũ | Đúng | Sửa nhiều | Sửa nhiều |

### Cách hoạt động

Mỗi field cần dịch → tạo 4 columns trong DB:

```
title       → proxy (trả giá trị theo ngôn ngữ hiện tại)
title_vi    → "Hợp Lực Banner"
title_en    → "Hop Luc Banner"
title_zh    → "合力横幅"
title_ko    → "합력 배너"
```

Admin hiển thị **tabs** cho mỗi ngôn ngữ:

```
┌── VI ──┬── EN ──┬── ZH ──┬── KO ──┐
│ Title: [Hợp Lực Banner]            │
│ Image: [banner-vi.png]             │
│ Alt:   [Hợp Lực Banner]            │
└─────────────────────────────────────┘
```

## API — Frontend gọi như thế nào

### Request

Frontend gửi `Accept-Language` header:

```
GET /api/pages/homepage/
Accept-Language: ko
```

Django `LocaleMiddleware` đọc header → activate ngôn ngữ → `modeltranslation` trả đúng fields.

### Response format — Flat (1 ngôn ngữ)

```json
{
  "banners": [
    {"title": "합력 배너", "image": "https://res.cloudinary.com/.../banner-ko.jpg", "alt": "합력 배너"}
  ],
  "about_section": {
    "label": "소개",
    "title": "합력",
    "description": "<p>순수 건설 분야에서 출발하여...</p>"
  }
}
```

Frontend chỉ nhận đúng ngôn ngữ đang chọn. Khi user switch ngôn ngữ → page reload (Next.js `[locale]` routing) → gọi API mới với header khác.

### Frontend integration (Next.js)

```typescript
// src/lib/api/client.ts
async function fetchAPI(endpoint: string, locale: string) {
  const res = await fetch(`${API_URL}${endpoint}`, {
    headers: { "Accept-Language": locale }
  });
  return res.json();
}

// src/app/[locale]/page.tsx
export default async function HomePage({ params: { locale } }) {
  const data = await fetchAPI("/api/pages/homepage/", locale);
  return <HomepageView data={data} />;
}
```

## Settings Django

```python
# config/settings/base.py

LANGUAGES = [
    ("vi", "Tiếng Việt"),
    ("en", "English"),
    ("zh", "中文"),
    ("ko", "한국어"),
]

LANGUAGE_CODE = "vi"

MODELTRANSLATION_DEFAULT_LANGUAGE = "vi"
MODELTRANSLATION_FALLBACK_LANGUAGES = {"default": ("vi",)}

MIDDLEWARE = [
    ...
    "django.middleware.locale.LocaleMiddleware",  # sau SessionMiddleware
    ...
]
```

## Admin — Tabs ngôn ngữ

Dùng `TabbedTranslationAdmin` từ `modeltranslation.admin`:

```python
from modeltranslation.admin import TabbedTranslationAdmin
from ordered_model.admin import OrderedModelAdmin

@admin.register(BannerSlide)
class BannerSlideAdmin(TabbedTranslationAdmin, OrderedModelAdmin):
    list_display = ["title", "image_preview", "is_active", "move_up_down_links"]
```

Thứ tự kế thừa: `TabbedTranslationAdmin` **trước** `OrderedModelAdmin`.

## Migration strategy

1. Thêm `django-modeltranslation` + config
2. Chạy `python manage.py makemigrations` → tạo migration thêm `_vi`, `_en`, `_zh`, `_ko` columns
3. Chạy `python manage.py migrate`
4. Chạy `python manage.py update_translation_fields` → copy data cũ sang `_vi`
5. Chạy `make seed.homepage.reset` → seed lại đủ 4 ngôn ngữ
